# 📚 Frameworks 

This folder maps your **home lab controls** to well‑known security frameworks and references:

* `iso-27001.md` — ISO/IEC 27001 Annex A control alignment
* `nist-csf.md` — NIST Cybersecurity Framework (CSF) alignment
* `soc2.md` — SOC 2 (Trust Services Criteria) alignment
* `mitre-attack.md` — MITRE ATT\&CK detections/use cases in the lab
* `irs-pub-1075.md` — IRS 1075 mapping (for reference only)

Use this section to show how your lab’s **policies, configurations, and validations** satisfy each framework’s intent. It’s not an audit deliverable — it’s a transparent engineering crosswalk.

---

## 🔎 How to Use This Folder

1. **Pick a framework page** (e.g., `nist-csf.md`).
2. For each function/category/control, link to:

   * The **control implementation**: `docs/network-plan.md`, `configurations/*`, OPNsense rules, Cloudflare Gateway policies.
   * The **validation evidence**: `lab/validation/evidence/*` generated by the [Validation Checklist](../../lab/validation/validation-checklist.md).
   * Any **runbooks** or tuning notes under `docs/`.
3. Mark the **status** for each control: `Planned | Implemented | Monitored`.

> Tip: keep evidence date‑stamped (e.g., `2025‑08‑22`) and reference **NL‑###** test IDs from the checklist.

---

## 🧩 Control ID & Evidence Conventions

* **Lab test IDs**: `NL-001`, `NL-002`, … (from validation checklist)
* **Evidence path**: `lab/validation/evidence/YYYY-MM-DD/<artifact>`
* **Config path**: `configurations/<system>/...`
* **Status**: `Planned`, `Implemented`, `Monitored` (has dashboard/alert coverage)

Front‑matter template you can paste at the top of each framework page:

```yaml
---
last_reviewed: 2025-08-22
owner: Matthew Levorson
status_legend:
  - Planned
  - Implemented
  - Monitored
---
```

## 🗂️ Suggested Structure Inside Each Page

For each control/reference:

1. **Control Objective** — in your own words.
2. **Implementation** — which system(s) and configuration snippets.
3. **Validation** — link NL‑### test(s) + evidence.
4. **Monitoring** — dashboards/alerts and responsible owner.
5. **Gaps / Next Steps** — concrete follow‑ups.

Example snippet (Markdown):

```md
### PR.AC-5 — Network integrity & segmentation
**Objective:** Prevent unauthorized lateral movement.
**Implementation:** OPNsense VLAN rules (deny RFC1918 from IoT/Guest); NAT DNS enforcement.
**Validation:** NL‑006, NL‑007, NL‑008 (evidence 2025‑08‑22).
**Monitoring:** Wazuh dashboard – blocked inter‑VLAN traffic widget; weekly review.
**Gaps:** Add explicit allowlist for vendor IoT endpoints.
```
